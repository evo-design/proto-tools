# NCBI Entrez Fetch

## Overview

`ncbi-fetch` provides a single-API-call interface to NCBI Entrez for searching and retrieving biological sequences (protein, nucleotide, gene) via `esearch`, `esummary`, and `efetch` operations.

## When to Use This Tool

**Primary use cases:**
- Searching NCBI databases by term (gene name + organism)
- Fetching sequences by accession/ID in FASTA format
- Retrieving gene summaries via esummary
- Subsequence extraction with coordinates and strand

**When NOT to use this tool:**
- Multi-source orchestrated fetches across NCBI, UniProt, and PDB: use `sequence-fetch`
- Sequence similarity search: use `blast-search`
- Large-scale homology search: use `mmseqs-search-proteins`

## How It Works

The tool wraps NCBI Entrez E-utilities (`esearch`, `esummary`, `efetch`) with automatic retry/backoff and session management. Each request specifies a database (`protein`, `nuccore`, `gene`), an operation, and either an identifier or search term.

## Quick Start

```python python icon="python"
from bio_programming_tools.tools.database_retrieval import (
    NCBIFetchConfig,
    NCBIFetchInput,
    run_ncbi_fetch,
)

# Search for lacI in E. coli
inputs = NCBIFetchInput(
    db="protein",
    operation="esearch",
    search_term="lacI[Gene Name] AND Escherichia coli[Organism]",
    max_results=5,
)

output = run_ncbi_fetch(inputs, NCBIFetchConfig())
print(output.operation, output.db, output.ids)
```
