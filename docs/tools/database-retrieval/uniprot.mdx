# UniProt Fetch

## Overview

`uniprot-fetch` retrieves protein entries from UniProt by accession ID or by searching with target name and organism. It returns the protein sequence, gene names, and PDB cross-references.

## When to Use This Tool

**Primary use cases:**
- Fetching a protein sequence by UniProt accession (e.g. P04637)
- Searching UniProt by gene name + organism to resolve accessions
- Extracting PDB cross-references from UniProt entries

**When NOT to use this tool:**
- Multi-source orchestrated fetches across NCBI, UniProt, and PDB: use `sequence-fetch`
- DNA or RNA sequence retrieval: use `ncbi-efetch`
- PDB structure metadata or chain sequences: use `pdb-fetch-entry` / `pdb-fetch-fasta`

## How It Works

The tool wraps the UniProt REST API. Entry lookups fetch directly by accession. Name-based searches use the UniProt search endpoint with organism filtering, then rank results by review status and gene name match quality.

## Quick Start

```python python icon="python"
from bio_programming_tools.tools.database_retrieval import (
    UniProtFetchConfig,
    UniProtFetchInput,
    run_uniprot_fetch,
)

# Fetch TP53 by accession
inputs = UniProtFetchInput(uniprot_id="P04637")

output = run_uniprot_fetch(inputs, UniProtFetchConfig())
print(output.accession, output.sequence[:20], output.length)
```

## Best Practices & Gotchas

1. Provide either `uniprot_id` or both `target_name` and `organism` — the validator enforces this.
2. Name-based search uses two query strategies (exact gene match, then broader gene+protein match) and returns the highest-ranked result.
3. Set `prefer_pdb_crossref=True` when you need both protein sequence and structure — this prioritizes entries with linked PDB IDs.
4. The `entry_type` field indicates review status: `UniProtKB reviewed (Swiss-Prot)` entries are manually curated and more reliable than `UniProtKB unreviewed (TrEMBL)` entries.
5. Use `raw_entry` for advanced use cases — it contains the full UniProt JSON record including function annotations, cross-references, and feature tables.

## Related Tools

- `sequence-fetch`: multi-source orchestrator across NCBI, UniProt, and PDB
- `ncbi-esearch` / `ncbi-esummary` / `ncbi-efetch`: NCBI Entrez searches and sequence retrieval
- `pdb-fetch-entry` / `pdb-fetch-fasta`: PDB structure metadata and chain sequences

## API Reference

### Input

<ParamField path="uniprot_id" type="string">
  UniProt accession for direct entry lookup
</ParamField>

<ParamField path="target_name" type="string">
  Gene or protein name for search
</ParamField>

<ParamField path="organism" type="string">
  Organism for search disambiguation
</ParamField>

<ParamField path="prefer_pdb_crossref" type="boolean" default="False">
  When searching, prefer entries that have linked PDB structures
</ParamField>

<ParamField path="max_candidates" type="integer" default="5">
  Maximum number of search results to evaluate when ranking
</ParamField>

### Configuration

<ParamField path="request_timeout_seconds" type="integer" default="15">
  HTTP timeout in seconds
</ParamField>

<ParamField path="http_retries" type="integer" default="2">
  Retries for HTTP requests
</ParamField>

<ParamField path="backoff_seconds" type="number" default="1.0">
  Seconds to wait between retries (doubles after each attempt)
</ParamField>

<ParamField path="user_agent" type="string" default="bio-programming-tools/uniprot-fetch-v1">
  Identifier string sent to database APIs with each request
</ParamField>

### Output

<ResponseField name="accession" type="string" required>
  Primary UniProt accession
</ResponseField>

<ResponseField name="sequence" type="string">
  Protein sequence
</ResponseField>

<ResponseField name="length" type="integer">
  Sequence length
</ResponseField>

<ResponseField name="entry_type" type="string">
  Review status (e.g. 'UniProtKB reviewed (Swiss-Prot)' for curated entries)
</ResponseField>

<ResponseField name="gene_names" type="List[string]">
  Gene name symbols
</ResponseField>

<ResponseField name="pdb_crossrefs" type="List[string]">
  PDB structure IDs linked to this protein entry
</ResponseField>

<ResponseField name="source_url" type="string" required>
  UniProt entry URL
</ResponseField>

<ResponseField name="raw_entry" type="Dict[string, any]">
  Complete UniProt JSON record for advanced programmatic access
</ResponseField>
